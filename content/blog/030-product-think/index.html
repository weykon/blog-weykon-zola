从我第一次去开发自己一个简单的、面向大众的、开源免费的软件 New Man.

处理每个简单的issue，一点点迭代，慢慢了解到从产品的角度去思考工作于软件的意义。

这个软件只能算上工具，除了一个窗口，没有太多的内容。

而同样功能的工具，有使用electron的chatGPT-mac, 和在mac app store上架的MenubarX.

MenubarX 似乎跟苹果所想做的有一个重叠的平台。

苹果的M1 mac，让电脑直接运行 Mobile App。 在他自家的swift生态中，直接将移动端的开发力量可以无逢转移到了桌面端。 MenubarX 将web归纳到随即随到的窗口，也是同样的想象，生态是web前端，更多的新鲜力量（初级且满血的年轻少年） 大家都喜欢随即随到，随想随即的效果。

在复杂度加深的开发实践中，从工具向的开发到融合了广阔的后端需求的应用

2023-01-25 又玩了一下VR后，有一些界面在结合AR下， AR应用又是一轮互联网APP的爆发浪潮 在结合先进开发效率，不再是如此贫乏的生产。

在程序员拥有足够多的时间、高的效率， 高通用型和便利自定义的高需求市场环境 来集合了不同人群的参与 就比如现在青年有接触大学教育都需要完成基本的电脑操作 PPT、Word 一样 以后的开发足以让大家的非常便利理解

  • GitHub 结合AI代码分析，从微软收购OpenAI，我们可以放大胆瞎想 类似于Copilot，让AI从功能性、功能块去区分、理解，整理出满足不同功能点的需求模型 GitHub 仓库模块化(Github Package)已经算为这项设想做下铺垫 配合程序员的描述，直接来完成代码包、构建包。 目前通过注释来自动填写代码，还是需要程序人较为专业的知识，还不够完成它最好的使命（AI会让它更完美）
