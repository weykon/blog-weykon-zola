Trait upcasting This release includes a long awaited feature — the ability to upcast trait objects. If a trait has a supertrait you can coerce a reference to said trait object to a reference to a trait object of > the supertrait:

trait Trait: Supertrait {} trait Supertrait {}

fn upcast(x: &dyn Trait) -> &dyn Supertrait { > x } The same would work with any other kind of (smart-)pointer, like Arc -> Arc or *const dyn Trait -> *const dyn Supertrait.

Previously this would have required a workaround in the form of an upcast method in the Trait itself, for example fn as_supertrait(&self) -> &dyn Supertrait, and this would work only for one kind > of reference/pointer. Such workarounds are not necessary anymore.

Note that this means that raw pointers to trait objects carry a non-trivial invariant: "leaking" a raw pointer to a trait object with an invalid vtable into safe code may lead to undefined > behavior. It is not decided yet whether creating such a raw pointer temporarily in well-controlled circumstances causes immediate undefined behavior, so code should refrain from creating such > pointers under any conditions (and Miri enforces that).

Trait upcasting may be especially useful with the Any trait, as it allows upcasting your trait object to dyn Any to call Any's downcast methods, without adding any trait methods or using external > crates.

use std::any::Any;

trait MyAny: Any {}

impl dyn MyAny { > fn downcast_ref(&self) -> Option<&T> { > (self as &dyn Any).downcast_ref() > } } You can learn more about trait upcasting in the Rust reference.

Trait 向上转换

这个版本包含了一个期待已久的特性 — 向上转换 trait 对象的能力。如果一个 trait 有一个父 trait，你可以将该 trait 对象的引用强制转换为父 trait 对象的引用：

trait Trait: Supertrait {}
trait Supertrait {}

fn upcast(x: &dyn Trait) -> &dyn Supertrait {
    x
}

同样的操作适用于任何其他类型的（智能）指针，如 Arc<dyn Trait> -> Arc<dyn Supertrait> 或 *const dyn Trait -> *const dyn Supertrait。

以前，这需要在 Trait 本身中添加一个向上转换方法作为变通方法，例如 fn as_supertrait(&self) -> &dyn Supertrait，而且这只适用于一种引用/指针类型。现在这些变通方法不再需要了。

请注意，这意味着指向 trait 对象的原始指针携带着一个非平凡的不变性："泄漏"一个带有无效 vtable 的 trait 对象的原始指针到安全代码中可能导致未定义行为。目前尚未确定在受控情况下临时创建此类原始指针是否会立即导致未定义行为，因此代码应避免在任何条件下创建此类指针（Miri 会强制执行这一点）。

Trait 向上转换对于 Any trait 特别有用，因为它允许将你的 trait 对象向上转换为 dyn Any 以调用 Any 的 downcast 方法，而无需添加任何 trait 方法或使用外部 crate。

use std::any::Any;

trait MyAny: Any {}

impl dyn MyAny {
    fn downcast_ref<T>(&self) -> Option<&T> {
        (self as &dyn Any).downcast_ref()
    }
}

你可以在 Rust 参考文档中了解更多关于 trait 向上转换的信息。

实用示例：为什么 Trait Upcasting 很有用

让我们通过一个更复杂的示例来展示这个特性的实用性：

use std::fmt::Debug;

// 基础trait定义基本行为
trait Animal: Debug {
    fn name(&self) -> &str;
    fn make_sound(&self) -> &str;
}

// 更专门化的子trait
trait Pet: Animal {
    fn owner(&self) -> &str;
}

// 具体实现
struct Dog {
    name: String,
    owner_name: String,
}

impl Animal for Dog {
    fn name(&self) -> &str {
        &self.name
    }
    
    fn make_sound(&self) -> &str {
        "Woof!"
    }
}

impl Pet for Dog {
    fn owner(&self) -> &str {
        &self.owner_name
    }
}

impl Debug for Dog {
    fn fmt(&self, f: &mut std::fmt::Formatter<'_>) -> std::fmt::Result {
        write!(f, "Dog {{ name: {}, owner: {} }}", self.name, self.owner_name)
    }
}

// 动物庇护所功能 - 只关心基本的Animal特性
fn describe_animal(animal: &dyn Animal) {
    println!("这是一只名叫{}的动物，它发出\"{}\"的声音", animal.name(), animal.make_sound());
}

// 宠物登记功能 - 需要Pet特性
fn register_pet(pet: &dyn Pet) {
    println!("登记宠物：{} - 主人：{}", pet.name(), pet.owner());
    
    // 这里可以直接将Pet向上转换为Animal以重用现有功能
    describe_animal(pet); // 无需任何显式转换!
}

fn main() {
    let dog = Dog {
        name: String::from("小黄"),
        owner_name: String::from("小明"),
    };
    
    // 可以将dog作为Pet引用
    let pet: &dyn Pet = &dog;
    
    // 现在可以直接将Pet向上转换为Animal
    let animal: &dyn Animal = pet;
    
    // 使用Animal的功能
    println!("动物信息: {:?}", animal);
    
    // 或者直接在函数中使用
    register_pet(pet);
}

主要优势

  1. 代码简化：不再需要手动实现转换方法，减少了模板代码。

  2. 通用性：适用于多种指针类型（&dyn, Box, Arc等），一次定义就能在所有地方使用。

  3. 灵活的类型系统：可以构建更符合直觉的类型层次结构，支持"is-a"关系。

  4. 与Any的整合：特别是与Rust的Any trait结合使用时，可以实现更强大的运行时类型识别功能，而不需要添加额外依赖。

  5. 代码重用：高级接口（如Pet）的用户可以无缝访问低级接口（如Animal）的功能，而不需要额外的转换代码。

通过这个特性，Rust进一步完善了其trait系统，使组件之间的交互和代码重用变得更加自然和直接。
