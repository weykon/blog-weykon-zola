WebGPU 阴影映射实现与调试指南

前言

在 WebGPU 中实现阴影映射是 3D 渲染中的一个重要环节，但也是容易出错的部分。本文将记录我在实现和调试 WebGPU 阴影映射过程中遇到的挑战，以及解决方案，特别关注顶点和索引缓冲区的管理问题。

阴影映射原理

阴影映射(Shadow Mapping)的基本思想是：

  1. 从光源视角渲染场景，记录深度信息到一个纹理（深度贴图）
  2. 从摄像机视角渲染场景，使用之前的深度贴图来判断每个片段是否在阴影中

我的实现架构

在我的项目中，阴影系统主要包含以下几个关键组件：

  1. DepthTexture - 负责创建和管理阴影深度纹理与采样器
  2. ShadowScene - 主要场景对象，包含几何体和渲染管线
  3. ShadowPlane - 地面平面，用于接收阴影

整体渲染流程分为两个通道：

  • 第一个通道：从光源角度渲染深度图
  • 第二个通道：渲染带阴影的主场景

遇到的问题：索引与顶点缓冲区管理

在实现过程中，我碰到的最大挑战是索引与顶点缓冲区的正确管理，这导致了很多难以调试的问题。

问题一：合并几何体时索引偏移错误

当我尝试将立方体和平面合并到同一个渲染通道时，忽略了需要为平面的索引添加适当的偏移量。

错误代码：

// 错误：直接合并索引，没有考虑顶点数组偏移
let for_common_index = cube.indices
    .iter()
    .cloned()
    .chain(plane.indices.iter().cloned())
    .collect::<Vec<u16>>();

正确做法应该是为平面的索引添加立方体顶点数量的偏移：

// 1. 获取立方体顶点数量
let cube_vertex_count = cube.vertices.len() as u16;

// 2. 应用偏移到平面索引
let plane_indices_with_offset = plane.indices
    .iter()
    .map(|idx| idx + cube_vertex_count) // 加上立方体顶点数作为偏移
    .collect::<Vec<u16>>();

// 3. 合并索引
let for_common_index = cube.indices
    .iter()
    .cloned()
    .chain(plane_indices_with_offset.iter().cloned())
    .collect::<Vec<u16>>();

问题二：顶点结构不一致

立方体使用的是包含位置、法线和颜色的复杂顶点结构，而平面使用的是简单的位置数组。合并时需要保持一致的格式。

解决方案是将所有几何体转换为一致的顶点格式：

// 提取立方体的位置信息
let for_depth_verticies = cube.vertices
    .iter()
    .map(|v| v.position)
    .collect::<Vec<[f32; 3]>>();

// 平面已经是位置格式
let plane_vertex = plane.vertices
    .iter()
    .map(|v| [v[0], v[1], v[2]])
    .collect::<Vec<[f32; 3]>>();

// 合并为一致的格式
let for_common_vertex = for_depth_verticies
    .iter()
    .cloned()
    .chain(plane_vertex.iter().cloned())
    .collect::<Vec<[f32; 3]>>();

深度纹理管理

另一个关键点是正确创建和使用深度纹理。在 WebGPU 中，如果要在一个通道渲染深度纹理，然后在另一个通道使用它，需要确保：

  1. 纹理有正确的用途标志
  2. 创建适当的视图
  3. 设置正确的采样器

// 创建视图
let shadow_view = shadow_texture.create_view(&wgpu::TextureViewDescriptor::default());

// 创建比较采样器
let shadow_sampler = device.create_sampler(&wgpu::SamplerDescriptor {
    // ...配置
    compare: Some(wgpu::CompareFunction::LessEqual),
    // ...
});

光源变换矩阵

在阴影映射中，我们需要两个重要的变换矩阵：

  1. 光源的视图投影矩阵 - 用于渲染深度图
  2. 相机的视图投影矩阵 - 用于主场景渲染

光源矩阵计算示例：

// 光源位置和目标
let light_position = Vec3::new(-3., 5., 3.);
let light_target = Vec3::ZERO;
let light_up = Vec3::Y;

// 创建视图矩阵
let light_view = Mat4::look_at_rh(light_position, light_target, light_up);

// 创建正交投影矩阵
let light_proj = Mat4::orthographic_rh(-3.0, 3.0, -3.0, 3.0, 0.1, 10.0);

// 组合成视图投影矩阵
let light_view_proj = light_proj * light_view;

着色器实现

阴影映射需要两个关键着色器：

1. 深度通道着色器

仅用于捕获深度信息，不需要输出颜色：

struct LightStorage {
    view_proj: mat4x4<f32>,
}
@group(0) @binding(0)
var<storage, read> light: LightStorage;

struct VertexInput {
    @location(0) position: vec3<f32>,
}
struct VertexOutput {
    @builtin(position) position: vec4<f32>,
}
@vertex
fn vs_main(in: VertexInput) -> VertexOutput {
    var out: VertexOutput;
    out.position = light.view_proj * vec4<f32>(in.position, 1.0);
    return out;
}

2. 主场景着色器

需要计算阴影并应用光照：

// 在顶点着色器中，计算光空间位置
@vertex
fn vs_main(in: VertexInput) -> VertexOutput {
    var out: VertexOutput;

    out.light_space_position = light.view_proj * vec4<f32>(in.position, 1.0);

阴影采样优化

为了获得更柔和的阴影边缘，我实现了 PCF (Percentage Closer Filtering) 阴影：

fn getShadow(coords: vec2<f32>, compare: f32) -> f32 {
    let texelSize = 1.0 / 1024.0;
    let bias = 0.02; 
    var shadow = 0.0;
    for (var y = -2; y <= 2; y += 1) {
        for (var x = -2; x <= 2; x += 1) {
            let offset = vec2<f32>(f32(x), f32(y)) * texelSize;
            shadow += textureSampleCompare(
                depth_texture, 
                shadow_sampler, 
                coords + offset, 
                compare - bias
            );
        }
    }
    return shadow / 25.0;
}

调试技巧

阴影映射的调试可能很困难，以下是我发现的一些有用的调试技巧：

  1. 可视化深度纹理 - 将深度纹理直接渲染到屏幕的一部分，帮助识别深度捕获问题
  2. 分开测试 - 先单独测试深度捕获通道，确保深度信息正确
  3. 使用明显的测试案例 - 使用简单、高对比度的场景进行测试
  4. 添加调试输出 - 使用 console_log! 打印关键数据
  5. 检查顶点和索引数量 - 确保数量匹配

总结与经验教训

实现 WebGPU 阴影映射的几个关键点：

  1. 正确管理索引偏移 - 合并几何体时，索引必须考虑顶点数组偏移
  2. 保持一致的顶点格式 - 所有几何体应使用一致的顶点格式
  3. 正确设置纹理用途 - 深度纹理需要同时支持渲染和采样
  4. 分离渲染通道 - 深度捕获和主场景渲染应在不同通道中完成
  5. 适当的深度偏移 - 使用深度偏移避免阴影痤疮（shadow acne）
  6. PCF 采样 - 使用 PCF 创建更柔和的阴影

通过这个项目，我深刻体会到了在 WebGPU 中精确管理图形资源的重要性，特别是顶点数据和索引缓冲区。虽然调试过程耗时，但这些经验对于掌握低级图形编程是非常宝贵的。
